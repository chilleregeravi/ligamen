    - Test: parseAgentOutput with JSON inside ```json block that has extra whitespace/newlines returns valid findings
    - Test: parseAgentOutput with raw JSON string (no fencing at all) returns valid findings
    - Test: parseAgentOutput with JSON embedded after prose text (no fencing) returns valid findings via raw parse
    - Test: parseAgentOutput with completely malformed text returns { valid: false } with error containing truncated preview (first 200 chars)
    - Test: parseAgentOutput with non-string input returns { valid: false }
    - Test: parseAgentOutput with JSON inside fenced block where JSON itself is invalid returns { valid: false } with parse error
  </behavior>
  <action>
    Modify parseAgentOutput in findings.js to implement a 3-strategy fallback chain:

    1. **Strategy 1 — Fenced code block**: Try existing regex `/```json\s*\n([\s\S]*?)\n```/` (current behavior). If match found, parse the captured group.
    2. **Strategy 2 — Raw JSON.parse**: Try JSON.parse(rawText.trim()). This handles cases where agent returns pure JSON with no markdown.
    3. **Strategy 3 — JSON substring extraction**: Find first `{` and last `}` in rawText, extract that substring, try JSON.parse. This handles prose-wrapped JSON without fencing.

    On each strategy: if JSON.parse succeeds, pass to validateFindings and return. If JSON.parse throws, move to next strategy.

    If ALL strategies fail: return `{ valid: false, error: "no parseable JSON in agent output (preview: " + rawText.slice(0, 200) + "...)" }`.

    The return type is unchanged: `{ valid: true, findings, warnings }` or `{ valid: false, error }`.

    Add tests to findings.test.js for the new strategies. Group them under a `describe("parseAgentOutput multi-strategy")` block.
  </action>
