<objective>
Extend `plugins/arcanon/worker/scan/enrichment/auth-db-extractor.js` with C# language
support: `AUTH_SIGNALS.csharp` covering ASP.NET Identity (`[Authorize]`, `AddAuthentication`,
`AddJwtBearer`, `AddDefaultIdentity`, `SignInManager`), `DB_SOURCE_SIGNALS.csharp` covering
EF Core with minimal-API `builder.Services.AddDbContext<T>()` pattern, `.cs` in
`LANG_EXTENSIONS`, and `'obj'` + `'bin'` added to `EXCLUDED_DIRS` so MSBuild output
directories are not traversed.

modern .NET 6+ services with `db_backend=null`. Without 'obj' + 'bin' in EXCLUDED_DIRS,
enrichment scans auto-generated AssemblyInfo.cs and TemporaryGeneratedFile_*.cs which
dilute signal.

concern in Phase 92, not this phase. Auth/DB detection works per-file; the body comparison
