proper version bracket, and stale rows from prior scans are cleaned up atomically.

  <action>
Replace the POST /scan try-block body (lines 183-195 of http.js) with:

```javascript
try {
  const repo = qe.upsertRepo({
    path: repo_path,
    name: repo_name || path.basename(repo_path),
    type: repo_type || "single",
  });
  const repoId = repo.id;
  const scanVersionId = qe.beginScan(repoId);
  try {
    qe.persistFindings(repoId, findings, commit || null, scanVersionId);
    qe.endScan(repoId, scanVersionId);
  } catch (innerErr) {
    // persistFindings failed — do NOT call endScan (bracket stays open / incomplete)
    // Re-throw so the outer catch logs and returns 500
    throw innerErr;
  }
  return reply.code(200).send({ status: "persisted", repo_id: repoId });
} catch (err) {
  httpLog('ERROR', err.message, { route: '/scan' });
  return reply.code(500).send({ error: err.message });
}
```

Key invariant: endScan is called ONLY when persistFindings succeeds. If persistFindings throws, the scan bracket row in scan_versions is left with completed_at=NULL (orphaned but harmless — it does not cause stale rows to be deleted). This is safer than calling endScan after a failed scan, which would delete valid prior-scan rows.
