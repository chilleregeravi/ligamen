<objective>
Fix THE-933: buildScanContext() returns a files list for incremental scans, but that file list is computed and then never mentioned in the agent prompt. The agent receives the full-scan prompt template with no mention of which files changed, so it re-scans the entire repo rather than focusing on the diff.

Purpose: Incremental scans are a performance optimization — the changed-files set must be communicated as a hard constraint to the agent, not just computed and discarded.

Output: scanRepos() injects a changed-files constraint section into the prompt for incremental scans. When the changed-files set is empty, the agent is told the scan is a no-op and agentRunner is not called — findings return as null with mode "incremental-noop".
</objective>

    ```javascript
    // The files list is advisory by default — we make it a hard constraint in the prompt.
    let finalPrompt = interpolatedPrompt;
    if (ctx.mode === "incremental" && ctx.files !== null) {
      const changedFiles = ctx.files.modified;

      // No-op: no files changed — skip agent invocation entirely
      if (changedFiles.length === 0) {
        slog('DEBUG', 'incremental-noop — no changed files', { repoPath });
        // Do NOT call beginScan — no bracket needed for a no-op
        results.push({ repoPath, mode: "incremental-noop", findings: null });
        continue;
      }

      const fileList = changedFiles.map(f => `  - ${f}`).join("\n");
      const constraint = [
        "",
        "---",
        "## INCREMENTAL SCAN — CHANGED FILES ONLY",
        "",
        "This is an incremental scan. You MUST only examine the following changed files.",
        "Do NOT read, analyze, or report connections from unchanged files.",
        "Scanning unchanged files wastes time and produces stale duplicate findings.",
        "",
        "Changed files:",
        fileList,
        "---",
      ].join("\n");
      finalPrompt = interpolatedPrompt + constraint;
    }
    ```
