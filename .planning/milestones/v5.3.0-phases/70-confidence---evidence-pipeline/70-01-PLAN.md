<objective>
Wire confidence and evidence from agent-validated findings through the upsert layer into the connections table, and project them in the /graph response.

Purpose: The agent already emits `confidence` and `evidence` per connection (validated by findings.js) but persistFindings() drops them — they are never written to the DB. Migration 009 (Phase 67) adds the columns. This plan closes the gap: write them on upsert, return them on getGraph().

Output:
- `query-engine.js`: `_stmtUpsertConnection` includes `confidence` and `evidence` columns (migration-009-aware fallback); `persistFindings()` passes `conn.confidence` and `conn.evidence`; `getGraph()` SELECTs `c.confidence, c.evidence` on connections
- `query-engine-confidence.test.js`: Node built-in test suite covering write and read-back of confidence/evidence
</objective>
