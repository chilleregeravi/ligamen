Create `plugins/ligamen/scripts/install-deps.sh` as an executable bash script. This script is invoked by the SessionStart hook (via hooks.json) and installs runtime npm deps into `${CLAUDE_PLUGIN_ROOT}`.

Follow the exact same non-blocking pattern as session-start.sh:
```
#!/usr/bin/env bash
# Ligamen — install-deps.sh
# SessionStart hook: installs MCP runtime dependencies into CLAUDE_PLUGIN_ROOT.
# Diff-based idempotency: skips if runtime-deps.json matches sentinel in CLAUDE_PLUGIN_DATA.
# Non-blocking: always exits 0.
set -euo pipefail
trap 'exit 0' ERR
```

Script logic (in order):

1. **Resolve CLAUDE_PLUGIN_ROOT**: Use env var if set, otherwise fall back to script-relative `$(cd "$(dirname "$0")/.." && pwd)`. Assign to `_R`.

2. **Guard: CLAUDE_PLUGIN_DATA must be set**: If `${CLAUDE_PLUGIN_DATA:-}` is empty, exit 0 silently (not running in plugin context — dev mode).

3. **Guard: jq must be available**: `command -v jq >/dev/null 2>&1 || exit 0`

4. **Guard: npm must be available**: `command -v npm >/dev/null 2>&1 || exit 0`

5. **Sentinel comparison (diff-based idempotency)**:
   - `SENTINEL="${CLAUDE_PLUGIN_DATA}/.ligamen-deps-installed.json"`
   - `MANIFEST="${_R}/runtime-deps.json"`
   - If `diff -q "$MANIFEST" "$SENTINEL" >/dev/null 2>&1` succeeds AND `[ -d "${_R}/node_modules/better-sqlite3" ]`, then exit 0 (deps already installed and valid).
   - This double-check covers both "deps changed" and "plugin update wiped node_modules" per user decision.

6. **Install deps**:
   - `npm install --prefix "${_R}" --omit=dev --no-fund --no-audit --package-lock=false 2>&1 | head -50 >&2`
   - Use `--prefix "${_R}"` to install into CLAUDE_PLUGIN_ROOT (per user decision — ESM walks up and finds node_modules).
   - npm reads the `package.json`-shaped `runtime-deps.json` — but since `--prefix` uses the directory's package.json, we need to copy `runtime-deps.json` to `"${_R}/package.json.bak"` first? No — per STACK.md, use `--package "${_R}/runtime-deps.json"` flag if available on npm 11+. If that flag is not available, copy runtime-deps.json to a temp dir and use `--prefix` on that temp dir. Actually, the simplest approach per user discretion: since runtime-deps.json IS a valid package.json shape, copy it to `${_R}/package.json` temporarily, run `npm install --prefix "${_R}" --omit=dev --no-fund --no-audit --package-lock=false`, then restore the original package.json. But that is fragile.

   Better approach: npm install reads the package.json in the --prefix directory. The existing `${_R}/package.json` has the same deps. So just run:
   ```bash
   npm install --prefix "${_R}" \
     --omit=dev --no-fund --no-audit --package-lock=false \
     2>&1 | head -50 >&2
   ```
   This installs the dependencies from `${_R}/package.json` (which contains the same runtime deps plus dev deps, but `--omit=dev` skips devDependencies). The runtime-deps.json serves as the sentinel for diffing, not as the npm manifest.

   Redirect stdout to stderr (all npm output must go to stderr — stdout must stay clean for hook JSON output if any).

7. **Check install result**:
   - If the `npm install` command (step 6) succeeds (exit 0):
     - Copy manifest to sentinel: `cp "$MANIFEST" "$SENTINEL"`
   - If it fails:
     - Delete partial node_modules: `rm -rf "${_R}/node_modules"`
     - Delete sentinel if it exists: `rm -f "$SENTINEL"`
     - Exit 0 (non-blocking)

8. **Final exit 0**.

Make the script executable: `chmod +x`.

IMPORTANT constraints:
- All output to stderr, never stdout (hook stdout must be clean JSON or empty)
- Always exit 0 regardless of failure (non-blocking per plugin convention)
- Use `trap 'exit 0' ERR` as first line after set
