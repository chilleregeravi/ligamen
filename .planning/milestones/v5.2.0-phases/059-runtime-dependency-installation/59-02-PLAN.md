**Part A: Extend mcp-wrapper.sh**

Rewrite `plugins/ligamen/scripts/mcp-wrapper.sh` to include self-healing dep installation before launching the MCP server. The wrapper is the last line of defense for the first-session race condition.

New content:
```bash
#!/usr/bin/env bash
# Ligamen — mcp-wrapper.sh
# Self-healing MCP server launcher.
# Checks for runtime deps before exec'ing server.js.
# If deps are missing (first-session race), installs them inline.
# All output to stderr — stdout must stay clean for MCP JSON-RPC.

_R="${CLAUDE_PLUGIN_ROOT:-}"
[ -z "$_R" ] && _R="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"

# Self-healing: if better-sqlite3 is missing, install deps before launching
if [ ! -d "${_R}/node_modules/better-sqlite3" ]; then
  echo "[ligamen] installing runtime deps (first run)..." >&2
  if command -v npm >/dev/null 2>&1; then
    npm install --prefix "${_R}" \
      --omit=dev --no-fund --no-audit --package-lock=false \
      2>&1 | head -50 >&2
    if [ $? -ne 0 ]; then
      echo "[ligamen] dep install failed — MCP server may not start" >&2
      rm -rf "${_R}/node_modules"
    fi
  else
    echo "[ligamen] npm not found — cannot install deps" >&2
  fi
fi

exec node "${_R}/worker/mcp/server.js"
```

Key constraints:
- ALL output (npm, echo) goes to stderr via `>&2` — stdout must be pristine for MCP JSON-RPC protocol
- The `$?` check must capture npm's exit code BEFORE the pipe to head. Use a subshell or process substitution to avoid masking the exit code. Actually, `set -o pipefail` would fix this but we don't want set -e here (wrapper should attempt exec even if install fails). Better: use a temp variable approach:
  ```bash
  npm install ... >"${_R}/.npm-install.log" 2>&1
  INSTALL_EXIT=$?
  head -50 "${_R}/.npm-install.log" >&2
  rm -f "${_R}/.npm-install.log"
  if [ $INSTALL_EXIT -ne 0 ]; then
  ```
- On install failure: clean up partial node_modules, print error, but still attempt exec (let Node give a clear error)
- No `set -e` or `trap` — the wrapper must always reach `exec node` even if install fails
- Keep the existing `_R` resolution logic unchanged

Make sure the script remains executable.
