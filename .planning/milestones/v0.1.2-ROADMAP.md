## Overview

Zero `ligamen` / `LIGAMEN` / `@ligamen` / `.ligamen` references anywhere in the plugin. The v0.1.1 "legacy honored for now" promise is retracted — Arcanon is the product name with no back-compat layer. Breaking change for any v5.x Ligamen upgrade path.

## Milestone Summary

**Key Decisions:**

- **Zero-tolerance policy on Ligamen refs** — No back-compat layers, no two-read fallbacks, no stderr deprecation warnings for legacy names. Just delete them. Rationale: leaving back-compat stubs permanently encodes the legacy name.
- **Breaking change accepted** — Users upgrading from v5.x must migrate their config file, data dir, and env vars. Migration instructions documented in CHANGELOG BREAKING section.
- **ChromaDB collection rename** — Existing `ligamen-impact` collections are orphaned on upgrade; users rebuild via `/arcanon:map`. Acceptable since ChromaDB is optional and rebuildable.
- **Wave-based parallel execution** — 101+102+104 ran in parallel; 103 depended on 101+102; 105 final gate. Enabled by design discipline (non-overlapping file sets).
- **Combined plan+execute for phases 102/103/104/105** — Scope was well-understood enough after 101 that spawning separate planners would have been ceremony. Saved ~4 agent round-trips.

**Issues Resolved:**

- 3 additional runtime ligamen reads discovered during 101 planning (pool.js, database.js, auth.js) — added as ENV-10, PATH-07/08/09 before execution.
- 9 residual cosmetic references found in Phase 105 sweep (test header comments, 1 test string literal) — cleaned inline before writing VERIFICATION.

**Known Carryovers (filed, not in v0.1.2 scope):**

- 2 pre-existing node test failures (`server-search.test.js` queryScan drift, `manager.test.js` incremental prompt mock) — confirmed unrelated to v0.1.2 via git-show against pre-101 base.
- 1 macOS-only HOK-06 p99 latency caveat from v0.1.1 — CI passes with `IMPACT_HOOK_LATENCY_THRESHOLD=100`.

**Stats:**
- 46/46 requirements complete

- Zero production regressions introduced
- Test suite: bats 309/310 green, node 524/526 green (with 2 pre-existing + 1 macOS caveat documented)
