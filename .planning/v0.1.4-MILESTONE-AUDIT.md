## 4. Tech Debt / Deferred Items

**RESCAN-PROD-01 — RESOLVED.** The original 118-02 / 119-01 implementations
drove rescan and shadow-scan through worker HTTP routes (`POST /api/rescan`,
`POST /scan-shadow`) that depended on an agent-runner injection inside the
worker process. Production worker startup never wires an agent runner —
Claude agents only run on the host via Claude Code — so both routes returned
`503 worker bootstrap incomplete` in production. Resolution (applied during
release prep, before v0.1.4 tag): both commands re-shaped as
**markdown-orchestrated** — the slash-command body invokes `Agent` directly
for discovery + deep scan, then persists via `openDb` (rescan) or
`getShadowQueryEngine` (shadow-scan) inline. Removed: the two HTTP routes,
`cmdRescan` / `cmdShadowScan` CLI handlers, `scanSingleRepo` wrapper, and
the `ARCANON_TEST_AGENT_RUNNER=1` test stub. Kept: `repo-resolver.js` (now
called inline from rescan.md), `getShadowQueryEngine` (shadow-scan
persistence target), Phase 117's `applyPendingOverrides` hook (now invoked
explicitly by the markdown command between `persistFindings` and `endScan`).
Cross-repo reconciliation preserved — rescan reads the live DB's existing
`services.name` set so single-repo rescan still classifies cross-service
connections correctly.

**Other deferred (lower priority):** 119-02 live↔shadow lock-key sharing; 120-01 persist `connections.line_start`/`line_end`; 120-01 hub-side v1.2 envelope adoption (separate repo); 121-01 catalog `--reload` flag. **No production blockers.**
