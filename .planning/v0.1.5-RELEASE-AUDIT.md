## Check 1 — cmdStatus PII leak

**Status:** 3 findings (1 BLOCKER, 2 POLISH)

The whole milestone's charter is "no `$HOME` paths leak from any worker egress seam." `/arcanon:status` is the only un-masked surface left.

### Findings

- **[BLOCKER]** `plugins/arcanon/worker/cli/hub.js:384` — `data_dir: resolveDataDir()` emits raw absolute path (`/Users/<name>/...`) into both stdout and `--json` output.
  - **Suggested fix**: wrap with `maskHome(resolveDataDir())`. Import `maskHome` from `../lib/path-mask.js`.

- **[BLOCKER]** `plugins/arcanon/worker/cli/hub.js:385` — `config_file: resolveConfigPath(process.cwd())` emits raw absolute path. Same impact: `/arcanon:status --json` payload contains `/Users/<name>/...`.
  - **Suggested fix**: same — wrap with `maskHome()`.

- **[BLOCKER]** `tests/pii-masking.bats` — no test asserts `/arcanon:status` output is leak-free. The whole point of pii-masking.bats is the structural regression guard; gap of this size in coverage means a future regression here would ship silently.
  - **Suggested fix**: add a 12th bats test `PII-bats-12: /arcanon:status --json contains no /Users/`.

- **[POLISH]** `plugins/arcanon/worker/cli/hub.js:404` — human-mode `data dir: ${report.data_dir}` line — same root cause; if line 384 is fixed (mask at source), this output flows through correctly.
  - **Suggested fix**: closes when 384 is fixed.

- **[POLISH]** `plugins/arcanon/worker/cli/hub.js:1424-1425` — `cmdDoctor` check 5 (`data_dir_writable`) calls `resolveDataDir()` and may surface the raw path in WARN/FAIL output. Out of v0.1.5 charter strictly, but adjacent.
  - **Suggested fix**: defensive — pass through `maskHome()` in any output emitted from doctor checks.
