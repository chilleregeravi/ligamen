Land services.base_path end-to-end in a single coherent change: migration 013 adds the column idempotently; the agent prompt and schema instruct the scanner to emit base_path; findings.js accepts it; persistFindings writes it; connection resolution strips target.base_path before path comparison so reverse-proxy-prefixed services no longer produce false-mismatch findings.

Today these show as "endpoint not exposed" mismatches; after this phase they correctly resolve.

    If your service is exposed under a URL prefix that the reverse proxy or ingress strips before forwarding, set `base_path` to that prefix. This lets connection resolution match `/api/users` (caller view) against your service's internal `/users` (handler view).

    **When to emit:**

      if (connPath.startsWith(bp + "/")) return connPath.slice(bp.length);
      // bp is a substring but not a path-segment boundary — D-03 case 4
      return null;
    }
