<objective>
Create the AllClear guard hook that hard-blocks writes to sensitive, lock, and generated-directory files via PreToolUse exit 2, and soft-warns on migration, generated-code, and CHANGELOG files via systemMessage JSON on stdout + exit 0. Register the hook in hooks.json and verify with a comprehensive bats test suite.

Purpose: This is the only blocking hook in AllClear -- it prevents Claude from accidentally overwriting credentials, lock files, and vendor directories. Getting the exit code semantics right (exit 2 = deny, exit 0 = allow) is the single most important correctness property.

Output: `scripts/file-guard.sh` (guard logic), `hooks/hooks.json` (registration), `tests/file-guard.bats` (verification)

Create `scripts/` directory and `scripts/file-guard.sh` following the architecture from 05-RESEARCH.md:

1. **Shebang and header**: `#!/usr/bin/env bash` -- no `set -e` (realpath can fail on new files; exit codes must be explicit).

2. **ALLCLEAR_DISABLE_GUARD check**: If env var is "1", exit 0 immediately.

3. **Read stdin once**: `INPUT=$(cat)` then extract file path via `jq -r '.tool_input.file_path // .tool_input.path // empty'`. If empty, exit 0 (not a file operation).

4. **Path normalization**: Use `realpath -m` with fallback for macOS:
   ```bash
   if command -v realpath &>/dev/null; then
     FILE=$(realpath -m "$RAW_FILE" 2>/dev/null || printf '%s' "$RAW_FILE")
   else
     FILE=$(cd "$(dirname "$RAW_FILE")" 2>/dev/null && printf '%s/%s' "$(pwd)" "$(basename "$RAW_FILE")" || printf '%s' "$RAW_FILE")
   fi
   BASENAME=$(basename "$FILE")
   ```

5. **Hard-block checks** (each sets BLOCK_REASON and calls block function that prints to stderr and exits 2):
   - Secret/credential files: `.env`, `.env.*`, `*.pem`, `*.key`, `*credentials*`, `*secret*`
   - Lock files: `*.lock`, `package-lock.json`, `bun.lock` (use case statement)
   - Generated directories: `*/node_modules/*`, `*/.venv/*`, `*/target/*`
   - ALLCLEAR_EXTRA_BLOCKED: Parse colon-separated env var, check each pattern against BASENAME and FILE

6. **Soft-warn checks** (each prints `{"systemMessage":"..."}` to stdout and exits 0):
   - Migration files: `*/migrations/*.sql`, `*/migrations/*.py`
   - Generated code: `*.pb.go`, `*_generated.*`, `*.gen.*`
   - CHANGELOG: `CHANGELOG.md`, `CHANGELOG`

7. **Default**: exit 0 (allow with no output)

The `block_file` function:
```bash
block_file() {
  local file="$1" reason="$2"
  printf 'AllClear: blocked write to %s — %s\n' "$(basename "$file")" "$reason" >&2
  exit 2
}
```

The `warn_file` function:
```bash
warn_file() {
  local file="$1" message="$2"
  printf '{"systemMessage": "AllClear: %s — %s"}\n' "$(basename "$file")" "$message"
  exit 0
}
```

Make the script executable: `chmod +x scripts/file-guard.sh`.

Key constraints from research:

- ALLCLEAR_EXTRA_BLOCKED env var: split on `:`, iterate patterns, check with bash glob `[[ "$BASENAME" == $pattern ]]` (unquoted pattern for glob expansion)
