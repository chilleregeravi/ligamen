  <name>Task 1: Create scripts/worker-start.sh with PID/port management and stale-PID detection</name>
  <files>scripts/worker-start.sh</files>
  <action>
Create `scripts/worker-start.sh` as an executable bash script. Follow the same header conventions as `scripts/session-start.sh` (shebang, set -euo pipefail, PLUGIN_ROOT resolution).

Implement this exact logic:

1. Resolve PLUGIN_ROOT from CLAUDE_PLUGIN_ROOT env var or script-relative path (same pattern as session-start.sh).

2. Determine DATA_DIR: `${ALLCLEAR_DATA_DIR:-$HOME/.allclear}`. Create it with `mkdir -p`.

3. Set PID_FILE="${DATA_DIR}/worker.pid" and PORT_FILE="${DATA_DIR}/worker.port".

4. Stale-PID check: if PID_FILE exists, read the PID. Run `kill -0 "$PID" 2>/dev/null`. If that succeeds, the worker is already running — print "worker already running (PID $PID)" to stdout and exit 0. If kill -0 fails, print "removing stale PID file (PID $PID no longer exists)" to stderr and rm -f PID_FILE.

5. Determine port:
   - First check ALLCLEAR_WORKER_PORT env var.
   - Else read from ~/.allclear/settings.json key "ALLCLEAR_WORKER_PORT" via jq (if the file exists).
   - Else check allclear.config.json key ."impact-map".port via jq (if file exists in CWD).
   - Default to 37888.

6. Write PORT_FILE immediately (before spawning): `echo "$PORT" > "$PORT_FILE"`. This allows the client library to read the port immediately after worker-start.sh exits.

7. Spawn the worker:
   ```bash
   nohup node "${PLUGIN_ROOT}/worker/index.js" \
     --port "$PORT" \
     --data-dir "$DATA_DIR" \
     >>"${DATA_DIR}/logs/worker.log" 2>&1 &
   WORKER_PID=$!
   echo "$WORKER_PID" > "$PID_FILE"
   ```
   Create the logs dir before spawning: `mkdir -p "${DATA_DIR}/logs"`.

8. Print "worker started (PID $WORKER_PID, port $PORT)" to stdout.

9. Exit 0.

Make the file executable: `chmod +x scripts/worker-start.sh`.

Do NOT wait for the worker to be ready in this script — readiness polling is in lib/worker-client.sh.

Use `jq` for JSON reading, but guard each jq call: if jq is not available or the file does not exist, fall through to the next source or default.

  <name>Task 2: Create scripts/worker-stop.sh with graceful SIGTERM and PID file cleanup</name>
  <files>scripts/worker-stop.sh</files>
  <action>
Create `scripts/worker-stop.sh` as an executable bash script.

Logic:

1. Resolve DATA_DIR the same way as worker-start.sh: `${ALLCLEAR_DATA_DIR:-$HOME/.allclear}`.

2. PID_FILE="${DATA_DIR}/worker.pid", PORT_FILE="${DATA_DIR}/worker.port".

3. If PID_FILE does not exist, print "worker is not running (no PID file)" and exit 0.

4. Read PID from PID_FILE.

5. Check if process is alive with `kill -0 "$PID" 2>/dev/null`. If not alive, print "worker is not running (PID $PID stale)", remove PID_FILE and PORT_FILE, exit 0.

6. Send SIGTERM: `kill -TERM "$PID"`.

7. Wait up to 5 seconds for the process to exit (poll with `kill -0 "$PID" 2>/dev/null` in a loop at 500ms intervals, max 10 iterations). If the process exits before the timeout, print "worker stopped (PID $PID)" and exit 0.

8. If process still running after 5s, send SIGKILL: `kill -9 "$PID"` and print "worker force-killed (PID $PID)" to stderr.

9. Remove PID_FILE and PORT_FILE with `rm -f`.

10. Exit 0.

Make the file executable: `chmod +x scripts/worker-stop.sh`.

Export three functions:

**worker_running():**
```bash
worker_running() {
  local data_dir="${ALLCLEAR_DATA_DIR:-$HOME/.allclear}"
  local port_file="${data_dir}/worker.port"
  [[ -f "$port_file" ]] || return 1
  local port; port=$(cat "$port_file")
  [[ -n "$port" ]] || return 1
  curl -s --max-time 1 "http://localhost:${port}/api/readiness" >/dev/null 2>&1
}
```
Returns 0 if the worker answers /api/readiness within 1 second, 1 otherwise.

**worker_call():**
```bash
worker_call() {
  local endpoint="$1"; shift
  local data_dir="${ALLCLEAR_DATA_DIR:-$HOME/.allclear}"
  local port_file="${data_dir}/worker.port"
  [[ -f "$port_file" ]] || { echo "worker-client: no port file at $port_file" >&2; return 1; }
  local port; port=$(cat "$port_file")
  [[ -n "$port" ]] || { echo "worker-client: port file is empty" >&2; return 1; }
  curl -sf --max-time 10 "http://localhost:${port}${endpoint}" "$@"
}
```
Passes any extra arguments to curl (e.g., -X POST, -d body).

**wait_for_worker():**
```bash
wait_for_worker() {
  local max_attempts="${1:-20}"
  local interval_ms="${2:-250}"
  local data_dir="${ALLCLEAR_DATA_DIR:-$HOME/.allclear}"
  local port_file="${data_dir}/worker.port"
  local i=0
  while [[ $i -lt $max_attempts ]]; do
    if worker_running; then
      return 0
    fi
    sleep "$(echo "scale=3; $interval_ms/1000" | bc)"
    i=$((i + 1))
  done
  echo "worker-client: timed out waiting for worker after $((max_attempts * interval_ms))ms" >&2
  return 1
}
```
Default: 20 retries at 250ms = 5 second total wait. Caller can override both.
