  <name>Task 1: Add GET /api/verify endpoint to worker HTTP server</name>
  <files>plugins/arcanon/worker/server/http.js</files>
  <action>
Add a new Fastify route `GET /api/verify` alongside the existing
`/api/version`, `/api/readiness`, `/graph`, `/api/impact` routes.

Per D-05 the endpoint accepts query parameters:
  - `project=<absolute-root-path>` (required) — resolved via existing getQE()
  - `connection_id=<integer>` (optional) — verify exactly one connection
  - `source_file=<relative-path>` (optional) — verify connections whose
    source_file matches (exact match if given a path, basename match if given
    a basename — detect: if value contains `/` use exact equality, otherwise
    use basename equality on `path.basename(c.source_file)`)
  - With neither connection_id nor source_file: verify ALL connections from
    the latest scan version (resolved by `MAX(scan_version_id)` from the
    `connections` table for the target repo).

Per D-03 enforce a hard cap: if the un-scoped query would return more than
1000 connections, respond with HTTP 200 but `truncated: true`, `total:
<count>`, `results: []`, and `message: "too many connections (N > 1000) —
scope with --source <path> or --connection <id>"`. The CLI maps this to
exit code 1 (Task 2 / Task 3).

Per D-02 the handler MUST NOT write anything: no upserts, no scan_versions,
no enrichment_log. Reads only.

Verdict computation (per connection — one helper function, e.g.
`computeVerdict(conn, projectRoot)`):
  1. Resolve absolute path: `path.resolve(projectRoot, conn.source_file)`.
  2. If `!fs.existsSync(absPath)` → verdict `moved`,
     `evidence_present: false`, `message: "source_file not found at recorded
     path"`, `snippet: null`.
  3. Read file with `fs.readFileSync(absPath, 'utf8')`. If read throws (EACCES
     etc.) → verdict `moved` with the error message in `message` (treat as
     "can't see the file" — same user remedy as moved).
  4. If `!conn.evidence` (null or empty after trim) → verdict `ok`,
     `evidence_present: false`, `message: "no-evidence-recorded"`,
     `snippet: null`. (Pre-Phase-109 connections — degraded but not failure.)
  5. Search for literal `conn.evidence` substring in the file content. If not
     found → verdict `missing`, `evidence_present: false`,
     `message: "evidence snippet not found in file"`, `snippet: null`,
     `line_start: null`, `line_end: null`.
  6. If found, compute `line_start` (1-indexed) by counting `\n` before the
     match index. `line_end` = `line_start + (snippet.match(/\n/g) ?? []).length`.
     Extract `snippet` = first 80 chars of the matched evidence (truncated
     for terminal display).
  7. If `conn.method` is set (truthy non-empty), check whether the matched
     snippet contains the method as a whole word (case-insensitive: e.g.,
     `/\bPOST\b/i`). If method recorded but NOT present in snippet → verdict
     `method_mismatch`, `evidence_present: true`, `snippet: <truncated>`,
     `message: "method '<method>' not found in evidence"`. Otherwise
     verdict `ok`, `evidence_present: true`, `snippet: <truncated>`,
     `message: null`.

Response shape (per D-05):
```json
{
  "results": [
    { "connection_id": <int>, "verdict": "ok"|"moved"|"missing"|"method_mismatch",
      "source_file": <string>, "line_start": <int|null>, "line_end": <int|null>,
      "evidence_present": <bool>, "snippet": <string|null>, "message": <string|null>,
      "method": <string|null> }
  ],
  "total": <int>,           // count BEFORE cap (so caller knows true size)
  "truncated": <bool>,      // true when total > 1000 and unscoped
  "scope": "all"|"connection"|"source"
}
```

Validation:
  - Missing `project` query param → reply.code(400) with
    `{ error: "missing required param: project" }`
  - `connection_id` provided but not a positive integer → 400 with
    `{ error: "invalid connection_id" }`
  - `connection_id` provided but no row found → reply.code(404) with
    `{ error: "no connection with id <N>" }`
  - QE resolution fails (project not indexed) → 404 with
    `{ error: "project not indexed: <root>" }`

For DB read use the existing `_db` access pattern via `qe._db.prepare(...).all(...)`,
mirroring how `getGraph()` queries connections. Recommended SQL:
```sql
-- All in latest scan:
SELECT c.id AS connection_id, c.source_file, c.method, c.path, c.evidence
FROM connections c
WHERE c.scan_version_id = (
  SELECT MAX(scan_version_id) FROM connections WHERE scan_version_id IS NOT NULL
)
AND c.source_file IS NOT NULL

-- By id:
SELECT id AS connection_id, source_file, method, path, evidence
FROM connections WHERE id = ?

-- By source path (exact):
SELECT id AS connection_id, source_file, method, path, evidence
FROM connections WHERE source_file = ?
  AND scan_version_id = (SELECT MAX(scan_version_id) FROM connections WHERE scan_version_id IS NOT NULL)
```

---
description: Re-read cited source files and verify the recorded evidence still exists. Read-only — never modifies scan data. Returns per-connection verdict (ok/moved/missing/method_mismatch) for one connection, one source file, or every connection in the latest scan.
argument-hint: "[--connection <id> | --source <path>] [--all] [--json]"

**Body sections:**

1. **# Arcanon Verify** — one-paragraph mission statement: "Re-read the cited
   source file and confirm the recorded evidence is still where the scan said
   it would be. Use this to detect stale scan data without running a full
   `/arcanon:map`."

2. **## When to use** — short bullet list:
   - After significant code changes, before trusting `/arcanon:impact` results
   - In CI to fail the build when scan drift exceeds threshold
   - After a teammate's PR merges, to spot-check whether your scan is stale

3. **## Usage**
   | Invocation | Behaviour |
   |---|---|
   | `/arcanon:verify` | Verify ALL connections in the latest scan (D-06). Capped at 1000; use `--source` to scope larger scans. |
   | `/arcanon:verify --connection <id>` | Verify exactly one connection by integer ID. |
   | `/arcanon:verify --source <path>` | Verify all connections whose `source_file` matches the given relative path. Basename match if no `/` in the value, exact match otherwise. |
   | `/arcanon:verify --json` | Emit machine-readable JSON instead of the human table. Combinable with the scoping flags above. |

4. **## Verdicts** — table documenting D-01 verbatim:
   | Verdict | Meaning | Recommended action |
   |---|---|---|
   | `ok` | Cited evidence is still present at the recorded location. | None — connection is trustworthy. |
   | `moved` | The recorded `source_file` no longer exists at that path. | Rescan with `/arcanon:map` (the file was renamed or deleted). |
   | `missing` | The file exists but the recorded evidence snippet is gone. | Re-run `/arcanon:map` (or, when v0.1.5 ships, `/arcanon:correct`). |
   | `method_mismatch` | Snippet present, but the cited HTTP method doesn't appear in it. | Spot-check — usually means the agent misclassified the method. |

5. **## Step 1 — Detect worker** (per impact.md pattern):
   ```bash
   source ${CLAUDE_PLUGIN_ROOT}/lib/worker-client.sh
   WORKER_UP=$(worker_running && echo "yes" || echo "no")
   ```
   If `WORKER_UP=no`, print:
   `Worker is not running. Start it with /arcanon:map (which boots the worker as a side effect), then re-run /arcanon:verify.`
   Exit. Do NOT start the worker from verify (verify is read-only).

6. **## Step 2 — Run the verify pass**
   ```bash
   bash ${CLAUDE_PLUGIN_ROOT}/scripts/hub.sh verify $ARGUMENTS
   ```
   Relay the script's output verbatim. The exit code propagates: 0 ok, 1
   non-ok or truncated, 2 invocation error.

7. **## Step 3 — Interpret the result**
   Brief guidance:
   - All `ok` → trust the graph. Move on.
   - Any `moved` → rescan. The graph cannot be repaired from drift alone.
   - Any `missing` → either rescan or wait for `/arcanon:correct` (v0.1.5).
   - Any `method_mismatch` → spot-check the cited connection in the detail
     panel; if you confirm the method is wrong, mark it for `/arcanon:correct`.

8. **## Performance notes** — D-03:
   - Each connection = one file read. ~200 connections complete in <2s on a warm cache.
   - Hard cap at 1000 connections per call. If exceeded, scope with `--source`
     or `--connection`.

9. **## Read-only guarantee** — D-02:
   - `/arcanon:verify` performs ZERO writes. The scan database, audit log, and
     scan_versions table are byte-identical before and after.
   - Safe to run in pre-commit, CI, or in a tight loop.

End with a footer pointing to `/arcanon:correct` (deferred to v0.1.5) as the
