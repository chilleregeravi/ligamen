  <action>
Create `skills/deploy-verify/SKILL.md` with the following structure. This is the complete skill — no supporting scripts are needed.

**YAML Frontmatter** (exactly these fields):
- `name: deploy-verify`
- `description:` — trigger phrases: "check deploy", "verify deployment", "compare expected state", "/allclear deploy", "cluster matches config". Make it clear this compares expected vs actual Kubernetes state.
- `version: 1.0.0`
- `allowed-tools: Bash`

**Shell injection line** (immediately after frontmatter):
```
kubectl available: !`command -v kubectl >/dev/null 2>&1 && echo "yes" || echo "no"`
```

**Steps section** with numbered steps:

1. **Check prerequisites.** If kubectl is "no", print exactly:
   `AllClear deploy: kubectl not available — skipping deploy verification`
   Then stop. Do not proceed. (Covers DPLY-04)

2. **Check cluster permissions.** Run `kubectl auth can-i get pods` to verify read access. If denied, print:
   `AllClear deploy: insufficient cluster permissions — check kubeconfig context`
   Then stop.

3. **Parse arguments.** Accept optional arguments:
   - First positional arg = environment name (default: look for "production" or "prod")
   - `--diff` flag = show full unified diff output (DPLY-05)

4. **Detect overlay path.** Search for kustomize overlays in order:
   - `k8s/overlays/<env>/`
   - `deploy/overlays/<env>/`
   - `kustomize/overlays/<env>/`
   - `overlays/<env>/`
   - `k8s/<env>/` (flat layout)
   - Root `kustomization.yaml` (single-env projects)
   If no kustomize overlay found, check for `Chart.yaml` or `helm/` directory for Helm.
   If neither found, print what locations were searched and stop gracefully.

5. **Compare expected vs actual state.** (Covers DPLY-01)
   - For kustomize: run `kubectl diff -k <overlay-path>; DIFF_EXIT=$?`
   - For Helm: try `helm diff upgrade <release> <chart> -f values.yaml` first; if helm-diff plugin is absent (`helm plugin list | grep diff`), fall back to `helm template <release> <chart> -f values.yaml | kubectl diff -f -; DIFF_EXIT=$?`
   - CRITICAL: Exit code 1 means diffs found (NOT an error). Exit code 0 means in sync. Exit code >1 means kubectl error. Handle all three cases explicitly.

6. **Extract image tags.** (Covers DPLY-02)
   - Expected: `kubectl kustomize <overlay-path> | grep 'image:' | awk '{print $2}' | sort -u`
   - Actual: `kubectl get pods -o jsonpath='{range .items[*]}{.metadata.name}{"\t"}{.spec.containers[0].image}{"\n"}{end}'`
   - Show a comparison table: service | expected image | actual image | status (match/mismatch)

7. **Summarize configmap state.** (Covers DPLY-03)
   If `kubectl diff` output contains configmap changes, highlight them separately. For targeted inspection: `kubectl get configmap <name> -n <namespace> -o yaml`

8. **Report results.** Show:
   - Overall status: IN SYNC or DRIFTED
   - Image tag comparison table
   - Configmap diff summary (if any)
   - If `--diff` flag was passed (DPLY-05): show full unified diff output from step 5
   - If no `--diff` flag: show summary of changed resources only

**Important style notes:**
- Use `set +e` or `|| true` patterns when running `kubectl diff` — do NOT let exit code 1 stop execution
- Never instruct Claude to apply changes — this skill is read-only verification
- Always respect namespaces from the kustomize overlay; do not default to "default" namespace silently
