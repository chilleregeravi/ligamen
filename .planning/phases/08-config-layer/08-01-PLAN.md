  <name>Task 1: Create lib/config.sh and allclear.config.json.example</name>
  <files>lib/config.sh, allclear.config.json.example</files>
  <behavior>
    - CONF-01: When allclear.config.json exists with a siblings array, sourcing lib/config.sh populates ALLCLEAR_CONFIG_SIBLINGS bash array with those paths
    - CONF-01: When allclear.config.json is absent, ALLCLEAR_CONFIG_SIBLINGS is an empty array
    - CONF-01: When allclear.config.json is malformed JSON, a warning goes to stderr and ALLCLEAR_CONFIG_SIBLINGS is empty
    - CONF-01: Both relative paths (../api) and absolute paths (/opt/repos/api) are preserved as-is in the array
    - Guard variable _ALLCLEAR_CONFIG_LOADED prevents double-loading when sourced multiple times
    - ALLCLEAR_CONFIG_FILE env var can override the config file path (default: allclear.config.json)
  </behavior>
  <action>
    Create `lib/config.sh` as a sourceable bash library:

    1. Use `#!/usr/bin/env bash` shebang (no set -euo pipefail — this is sourced, not executed directly)
    2. Guard against double-source: `if [[ -n "${_ALLCLEAR_CONFIG_LOADED:-}" ]]; then return 0; fi`
    3. Set `_ALLCLEAR_CONFIG_LOADED=1`
    4. Set `ALLCLEAR_CONFIG_FILE="${ALLCLEAR_CONFIG_FILE:-allclear.config.json}"`
    5. Initialize `ALLCLEAR_CONFIG_SIBLINGS=()` (empty array)
    6. If `$ALLCLEAR_CONFIG_FILE` exists:
       - Use `while IFS= read -r path` loop (NOT mapfile — bash 3.2 compat per research pitfall 3)
       - Read from `jq -r '.siblings[]? // empty' "$ALLCLEAR_CONFIG_FILE" 2>/dev/null`
       - Skip empty lines with `[[ -n "$path" ]]` check
       - Append to ALLCLEAR_CONFIG_SIBLINGS array
       - If jq fails (malformed JSON), the `2>/dev/null` on jq plus the empty loop means ALLCLEAR_CONFIG_SIBLINGS stays empty
       - Add a separate malformed JSON detection: test with `jq '.' "$ALLCLEAR_CONFIG_FILE" >/dev/null 2>&1` and if it fails, emit warning to stderr: "allclear: warning: allclear.config.json is malformed, using defaults"
    7. Do NOT export the array (bash cannot export arrays across subshells) — consumers source this file
    8. This file sources nothing else (leaf node in source graph per research pitfall 5)

    Create `allclear.config.json.example`:
    ```json
    {
      "siblings": [
        "../api",
        "../ui",
        "../sdk"
      ]
    }
    ```
    Include a comment header is not possible in JSON, so keep it clean. The example file IS the documentation.
  </action>
  <verify>
    <automated>bats tests/config.bats --filter "config.sh"</automated>
  </verify>
  <done>lib/config.sh loads siblings from JSON when present, returns empty when absent, warns on malformed JSON. allclear.config.json.example documents the schema.</done>
