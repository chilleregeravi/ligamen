Three states:
- **State A — No worker, no map**: `WORKER_UP=no` (do not attempt to start worker here — cross-impact is a query command, not the orchestrator). Jump to "Legacy Fallback" section.
- **State B — Worker up, no map data**: Print "No scan data found. Run `/allclear:map` to build the dependency map first." — then jump to "Legacy Fallback" (still provide grep results as a partial answer).
- **State C — Worker up, map has data**: Proceed with graph query flow below.

**Usage section** (place after frontmatter, before steps):
- `/allclear:cross-impact` — auto-detect uncommitted changes and query impact
- `/allclear:cross-impact <symbol>` — query impact for a specific endpoint or service name
- `/allclear:cross-impact --changed` — same as no-args (git diff auto-detect)
- `/allclear:cross-impact --exclude <repo>` — exclude a repo from results

---

**State C: Graph Query Flow**

**Step 1: Detect changes**

If no symbol argument and no `--changed` flag:
- Run `git diff --name-only HEAD` for uncommitted changes.
- Run `git diff --name-only HEAD~1 HEAD` for the most recent commit.
- Combine and deduplicate. If nothing found, ask: "No changes detected. What would you like to check? (enter a service name or endpoint path)".

If symbol argument provided: use that directly as the query target.

**Step 2: Query the impact map**

For each changed file or symbol:
- Call `worker_call GET /impact?change=<endpoint_or_service>`.
- Collect all `affected` items from all responses.

Also run the transitive query: call `worker_call GET /impact?change=<target>&transitive=true` to get the full blast radius.

**Step 3: Render impact report**

Produce a structured report grouped by severity:

```
Impact Report — <date>
Changes detected: <N files changed>

CRITICAL — Endpoint removed (N services affected)
  auth-service  [consumers: api-gateway, user-service]
    └─ POST /auth/token removed
       Consumers will break at runtime.
       Files to update: src/client/auth.ts:42, src/gateway/auth-proxy.ts:18

WARN — Field type changed (N services affected)
  user-service  [consumers: reporting-service]
    └─ UserProfile.email: string → string | null
       Existing consumers may fail null checks.

INFO — Additive change (N services)
  user-service
    └─ UserProfile.avatar_url: string (new field)
       No existing consumers affected.

Transitive blast radius: <total unique services across all hops>
```

If no affected services: print "No impact found. Changes appear safe. (Verify the map is up to date.)"

**Step 4: Suggest re-scan if map may be stale**

After the report, check: are there uncommitted changes in any repo that has a `last_scanned_commit` earlier than the current HEAD?

If potentially stale: print:
```
Note: Your dependency map may not reflect recent code changes.
Run `/allclear:map` to re-scan and catch any new or removed connections.
```
Ask: "Re-scan now? (yes/no)" — if yes, print "Run `/allclear:map` to trigger a scan." (do not trigger scan inline — map is the orchestrator).
