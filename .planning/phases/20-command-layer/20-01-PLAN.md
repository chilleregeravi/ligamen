Create `commands/map.md` as a Claude command markdown file that implements the complete `/allclear:map` orchestration flow described in `.planning/designs/cross-impact-v2.md`. The command is a prompt that Claude executes — write it in imperative second-person ("Check...", "If...", "Spawn...") matching the style of `commands/cross-impact.md`.

Frontmatter:
```yaml
---
description: Build or refresh the service dependency map by scanning linked repos. Use when the user runs /allclear:map to build the impact map for the first time or re-scan after changes. Also handles --view to open the graph UI and --full to force a complete re-scan.
allowed-tools: Bash, Read, Write, AskUserQuestion
argument-hint: "[--view] [--full]"
---
```

Command body must implement this exact flow (match the flowchart in cross-impact-v2.md):

**Step 0: Check --view flag**
- If `--view` flag is present: check if map data exists via `worker_call GET /graph` (after ensuring worker is running). If data exists, open browser with `open http://localhost:$(cat ~/.allclear/worker.port)` and exit. If no data, print: "No map data yet. Run `/allclear:map` to build the dependency map first." and exit.

**Step 1: Ensure worker is running**
- Source `${CLAUDE_PLUGIN_ROOT}/lib/worker-client.sh`.
- If `worker_running` is false: run `bash ${CLAUDE_PLUGIN_ROOT}/scripts/worker-start.sh`, then `wait_for_worker`. If worker fails to start, show the error and exit.

**Step 2: Discover repos**
- Read `allclear.config.json` if present and extract `linked-repos`.
- Also run `source ${CLAUDE_PLUGIN_ROOT}/lib/linked-repos.sh && list_linked_repos` to find repos in parent directory and memory.
- Combine and deduplicate. Mark repos NOT in `allclear.config.json` as "(newly discovered)" in the display.

**Step 3: Present repo list and confirm**
- Print the combined list to the user. Ask: "Confirm these repos for scanning? (yes / edit / no)". Use `AskUserQuestion`.
- If "edit": ask for the updated list. Repeat.
- If "no": abort — print "Map build cancelled." and exit.
- Once confirmed: write `allclear.config.json` with `linked-repos` set to the confirmed list.

**Step 4: Determine scan mode**
- If `--full` flag: mode = full.
- Else: check `repo_state` via `worker_call GET /graph` to see if any repo has a `last_scanned_commit`. If no repo has been scanned before, mode = full (first scan). Otherwise mode = incremental.

**Step 5: Trigger scan**
- Call `worker_call POST /scan '{"repos": [...], "full": <bool>}'`.
- Print: "Scanning N repos... (this may take a few minutes)".
- Parse the JSON response — it contains findings grouped by confidence: `{"high": [...], "low": [...]}`.

**Step 6: Present findings and confirm**
- High-confidence findings: show a grouped summary. Example: "Found 12 services, 47 connections across 3 repos. (details below)". Ask: "Confirm and save these findings? (yes / edit / no)".
- Low-confidence findings (cap shown at 10): for each, ask a specific clarifying question from the finding's `clarification_question` field.
- If "edit": ask what to change and re-present the modified findings.
- If "no" to all: print "Scan cancelled. No data written." and exit.

**Step 7: Snapshot existing map (if re-scan)**
- Check `worker_call GET /versions`. If there is an existing map version (not empty list), ask: "Keep history snapshot before overwriting? (yes/no)". If yes, call `worker_call POST /scan/snapshot`.

**Step 8: Persist confirmed findings**
- Call `worker_call POST /scan/confirm` with the confirmed findings JSON.
- Print: "Dependency map saved."

**Step 9: Handle first-time build**
- Check if this was the first map build: the `/versions` response was empty before the persist in Step 7.
- If first build: update `allclear.config.json` to add the `impact-map` section: `{"linked-repos": [...], "impact-map": {"history": true}}`.
- Print MCP registration instructions:
  ```
  Map built successfully.

  To enable impact checking in all your Claude Code agents, add the AllClear MCP server
  to your project's .mcp.json:

  {
    "mcpServers": {
      "allclear-impact": {
        "type": "stdio",
        "command": "node",
        "args": ["${CLAUDE_PLUGIN_ROOT}/worker/mcp-server.js"]
      }
    }
  }

  Then restart Claude Code. Agents will automatically check impact before modifying services.
  ```

**Step 10: Open graph UI**
- Run `open http://localhost:$(cat ~/.allclear/worker.port)` to open the graph UI in the browser.
- Print: "Graph UI opened at http://localhost:PORT".

Include a "Usage" section at the top after frontmatter:
- `/allclear:map` — discover repos, scan, confirm, persist, view graph
- `/allclear:map --view` — open graph UI without scanning
- `/allclear:map --full` — force full re-scan (ignores git diff)
