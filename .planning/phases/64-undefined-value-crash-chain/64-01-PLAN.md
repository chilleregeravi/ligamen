<objective>
Add a `sanitizeBindings()` helper to QueryEngine that converts any `undefined` values to `null` in the binding object, then apply it in `upsertService` and `upsertConnection` before calling `.run()`.

Purpose: better-sqlite3 throws `TypeError: Cannot bind value: undefined` when any field in the binding object is JavaScript `undefined`. Agent scan output can contain fields that are absent (not even `null`) when optional manifest keys are missing, causing the scan to crash mid-write and triggering the CLI fallback path.

Output: Patched `query-engine.js` where no `.run()` call in upsertService or upsertConnection can ever receive an undefined binding.
</objective>

    1. Add a module-level (or class-level) helper function immediately before the QueryEngine class definition:

    ```js
    /**
     * Converts any `undefined` values in a binding object to `null`.
     * better-sqlite3 throws TypeError for undefined bindings; null is always safe.
     * @param {object} obj
     * @returns {object}
     */
    function sanitizeBindings(obj) {
      const out = {};
      for (const key of Object.keys(obj)) {
        out[key] = obj[key] === undefined ? null : obj[key];
      }
      return out;
    }
    ```

    2. Patch `upsertService` to wrap the binding object:
    ```js
    upsertService(serviceData) {
      const result = this._stmtUpsertService.run(
        sanitizeBindings({ type: "service", scan_version_id: null, ...serviceData })
      );
      return result.lastInsertRowid;
    }
    ```

    ```js
    upsertConnection(connData) {

    ```
