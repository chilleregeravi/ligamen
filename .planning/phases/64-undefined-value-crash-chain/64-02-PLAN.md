<objective>
Fix the CLI fallback scan in `commands/map.md` Step 4 so it passes the captured project root path to `openDb()` instead of relying on `process.cwd()`.

Purpose: `openDb()` computes the database path as `sha256(projectRoot).slice(0,12)`. When the CLI fallback runs `node -e` without an explicit projectRoot argument, `process.cwd()` may differ from the project directory Claude was scanning, producing a different hash and writing results to a phantom database file that the HTTP server never reads.

Output: Updated `map.md` where the Step 4 node snippet substitutes `$PROJECT_ROOT` (captured in Step 1 when the user confirmed repos) into `openDb('${PROJECT_ROOT}')`.
</objective>
