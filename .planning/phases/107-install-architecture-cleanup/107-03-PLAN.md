<objective>
Land the bats test suite for the rewritten install architecture (INST-07..11) plus an integration smoke for INST-12. Update `tests/mcp-wrapper.bats` to drop the deleted self-heal assertion (transient breakage from Plan 107-01).

**5 INST-07..11 scenarios (REQUIREMENTS.md):**

- **INST-07** happy-path-skip: sentinel matches AND binding loads → `install-deps.sh` exits in <100ms with no npm process spawned. Asserted via process-stub-marker AND latency measurement.
- **INST-08** broken-binding: delete `node_modules/better-sqlite3/build/Release/` → install-deps.sh detects via `validate_binding()` and triggers `npm rebuild better-sqlite3` once → binding loads after.
- **INST-09** prebuild-silent-fail: stub `npm install` that "succeeds" but leaves `build/Release/` empty → install-deps.sh detects via `require()` and triggers rebuild.
- **INST-10** fresh-install: empty `node_modules/`, no sentinel → install-deps.sh runs the full install path, validates binding, writes sentinel, exits 0.
- **INST-11** sentinel-mismatch: sentinel contains a bogus hex string different from `package.json` hash → install-deps.sh runs `npm install`, validates binding, writes the canonical hash to the sentinel.

**INST-12 integration smoke:**

The original INST-12 wording is: "No regression in fresh-install flow: `claude plugin marketplace add` + `claude plugin install` + first session start → worker daemon healthy, MCP server starts, slash commands work."

This requires a working Claude Code installation, a writable `~/.claude/plugins/marketplaces/` cache, and live npm. Two options:

A) **Try real `make install` in bats** — `tests/install-deps.bats` adds an INST-12 test that calls `make -f $PROJECT_ROOT/Makefile install` and asserts the install completes. This may fail on CI runners that lack `claude` in PATH. If it fails with "claude not found", the test SKIPs (bats `skip` directive); if it fails for any other reason, the test FAILs.

B) **Document INST-12 as a manual smoke** — the SUMMARY.md captures the manual smoke checklist, and Phase 113 (Verification Gate, VER-05) covers the actual fresh-install integration on Node 25.

This plan implements BOTH: an INST-12 bats test that auto-skips when `claude` is unavailable (preserving local-developer signal), AND a SUMMARY-level documentation entry that hands the manual smoke off to Phase 113.

**Purpose:** Without these tests, INST-04's "<100ms" requirement and INST-03's "rebuild fallback" requirement are unverifiable claims. The bats suite is the executable spec for the install architecture. Once green, Plans 107-01 + 107-02 + this plan together close all 12 INST requirements.

**Why this is a single wave-2 plan (not fan-out):**

The 5 scenarios share heavy setup infrastructure (mock plugin root, stub npm, stub node-or-real-node-with-fixture, hash precomputation). A single bats file with shared `setup()` and `teardown()` is more maintainable than 5 separate files. Per CONTEXT.md sizing, this fits a single ~30%-context plan because the test additions follow a repetitive structure (each test is ~20-40 lines of similar shape).

Key implementation details:

**1. Common setup pattern (in `setup()`):**
- `mktemp -d` for `MOCK_PLUGIN_ROOT` and `MOCK_PLUGIN_DATA`
- Copy `install-deps.sh` into the mock plugin root's `scripts/` dir
- Write a fixed `package.json` to the mock root with `better-sqlite3` as the actual dep (so tests can validate against a real installed binding)
- Pre-compute the canonical hash via the same `jq -c -S '.deps + .optDeps' | shasum -a 256 | awk` pipeline the script uses, store in `$EXPECTED_HASH` for sentinel-write/match scenarios
- Set up the npm-stub directory `STUB_NPM_DIR` with a stub that records invocation in `$NPM_INVOKED_MARKER`

**2. Real binding fixture pattern (for INST-08, INST-09, INST-10):**

The mock plugin root needs an actual `node_modules/better-sqlite3` for the `validate_binding` step to succeed. Three options:

a) Symlink: `ln -s "$PROJECT_ROOT/plugins/arcanon/node_modules" "$MOCK_PLUGIN_ROOT/node_modules"`. Fastest. Works if the project has been `npm install`-ed.

b) Copy: `cp -R "$PROJECT_ROOT/plugins/arcanon/node_modules" "$MOCK_PLUGIN_ROOT/node_modules"`. Slow, but isolation-safe. Use for tests that mutate node_modules (INST-08 deletes `build/Release/`).

c) Real install: `npm install --prefix "$MOCK_PLUGIN_ROOT"`. Slowest. Used only by INST-10 (fresh-install) which specifically tests the install path.

The bats suite uses (b) for INST-08 / INST-09 (mutation-safe), (c) for INST-10 only, and (a) for INST-07 / INST-11 (read-only). This balances speed and isolation.

**3. Latency measurement pattern (INST-07):**

```bash
# Use EPOCHREALTIME (bash 5+) for microsecond resolution
START="${EPOCHREALTIME}"
CLAUDE_PLUGIN_ROOT="$MOCK_PLUGIN_ROOT" \
  CLAUDE_PLUGIN_DATA="$MOCK_PLUGIN_DATA" \
  bash "$MOCK_PLUGIN_ROOT/scripts/install-deps.sh"
END="${EPOCHREALTIME}"
ELAPSED_MS=$(awk "BEGIN { printf \"%d\", ($END - $START) * 1000 }")

# Threshold: 100ms locally, 500ms on CI
if [[ -n "${CI:-}" ]]; then
  THRESHOLD_MS=500
else
  THRESHOLD_MS=100
fi

[[ "$ELAPSED_MS" -lt "$THRESHOLD_MS" ]]
```

If `EPOCHREALTIME` is unavailable (Bash < 5), fall back to `python3 -c 'import time; print(int(time.time()*1000))'`. macOS Bash 4.x is the dev-machine concern — Homebrew bash 5+ resolves this; the test docs recommend Homebrew bash for the test runner.

**6. INST-09 prebuild-silent-fail fixture:**

```bash
# Setup: empty node_modules + missing sentinel
rm -rf "$MOCK_PLUGIN_ROOT/node_modules"
rm -f "$MOCK_PLUGIN_DATA/.arcanon-deps-sentinel"

# Stub npm install: pretends to install but leaves build/Release/ empty
# Stub npm rebuild: actually rebuilds (delegate to real npm)
STUB_NPM_DIR=$(mktemp -d)
cat > "$STUB_NPM_DIR/npm" <<EOF
#!/usr/bin/env bash
case "\$1" in
  install)
    # Simulate prebuild-install silent failure: copy real node_modules but
    # wipe build/Release/ to mimic a broken-prebuild state
    cp -R "$PROJECT_ROOT/plugins/arcanon/node_modules" "\$2-tmp" 2>/dev/null || true
    if [[ "\$2" == "--prefix" ]]; then PREFIX="\$3"; else PREFIX="\$2"; fi
    cp -R "$PROJECT_ROOT/plugins/arcanon/node_modules" "\${PREFIX}/node_modules"
    rm -rf "\${PREFIX}/node_modules/better-sqlite3/build/Release"
    exit 0
    ;;
  rebuild)
    # Delegate to real npm rebuild
    exec /usr/bin/env npm "\$@"
    ;;
  *)
    exec /usr/bin/env npm "\$@"
    ;;
esac
EOF
chmod +x "$STUB_NPM_DIR/npm"

# Run
PATH="$STUB_NPM_DIR:$PATH" \
  CLAUDE_PLUGIN_ROOT="$MOCK_PLUGIN_ROOT" \
  CLAUDE_PLUGIN_DATA="$MOCK_PLUGIN_DATA" \
  bash "$MOCK_PLUGIN_ROOT/scripts/install-deps.sh"

# Assert: rebuild path engaged → build/Release/ exists post-rebuild
[[ -d "$MOCK_PLUGIN_ROOT/node_modules/better-sqlite3/build/Release" ]]
[[ -f "$MOCK_PLUGIN_DATA/.arcanon-deps-sentinel" ]]
```

Note: this stub is intentionally complex because INST-09 specifically tests "npm install reports success but binding broken". The stub must satisfy that contract exactly. The real `npm rebuild` delegation requires `/usr/bin/env npm` in PATH (the tests `setup()` doesn't pollute the original PATH). If `npm rebuild` itself can't be delegated (e.g., on a CI without C compiler), the test SKIPs.
