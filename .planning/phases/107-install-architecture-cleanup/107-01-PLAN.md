  ```bash
  # Self-healing: if better-sqlite3 is missing, install deps before launching
  if [ ! -d "${_R}/node_modules/better-sqlite3" ]; then
    echo "[arcanon] installing runtime deps (first run)..." >&2
    if command -v npm >/dev/null 2>&1; then

      fi
    else

    fi
  fi
  ```

**Specific changes versus the current file:**

- Variable name: `_R` → `PLUGIN_ROOT` (more readable; matches the convention used in install-deps.sh per Plan 107-02).
- Strict mode added: `set -euo pipefail` (the current file lacks this — it relies on `exec` to bubble up failures, but adding strict mode is consistent with every other script in `plugins/arcanon/scripts/`).

**Specific things preserved:**

- `CLAUDE_PLUGIN_ROOT` env-var resolution (the env var is set by Claude Code when the plugin is installed; the `BASH_SOURCE` fallback covers dev mode where the script is invoked directly).
- `${BASH_SOURCE[0]}` (NOT `${0}`) for the fallback resolution — `BASH_SOURCE` is correct when the script is sourced (defensive; the script isn't sourced today but the pattern matches lib/*.sh conventions).
- Final `exec node "${PLUGIN_ROOT}/worker/mcp/server.js"` — exec ensures node replaces the bash process so signals (SIGTERM from Claude Code) reach node directly. Same as the current line 30.

**Why no `trap 'exit 0' ERR`:**

Per CONVENTIONS.md, hook scripts use the trap because they emit `hookSpecificOutput` JSON and must never block Claude. mcp-wrapper.sh is NOT a hook — it's the MCP server launcher. If the launcher fails (e.g., `node` not in PATH), Claude Code legitimately surfaces the error to the user via the MCP transport. Swallowing those errors with a trap would make MCP server failures invisible. Same reasoning as why session-start.sh has `trap 'exit 0' ERR` but worker-start.sh does not.
