**FLAG 5 — MCP handshake decision: Option B (liveness probe).**

The original draft sent a raw `tools/list` JSON-RPC request without an `initialize` handshake, which violates the MCP protocol. Two options were available:
- **Option A:** Implement the full handshake (`initialize` → `notifications/initialized` → `tools/list`), parse three responses, count tools, expect ≥ 9. More faithful smoke; requires reading `worker/mcp/server.js` to confirm the exact handshake shape and ~30 lines of buffer-parsing.
- **Option B (CHOSEN):** Spawn the MCP server, wait up to 1 second for ANY valid JSON-RPC line on stdout (proves the server reached the message loop without crashing on import), kill it. Simpler, weaker, but still verifies "MCP server starts cleanly without crashing on import" — which is the actual failure mode operators care about (a missing dep, a bad import, a syntax error in mcp/server.js itself).

Rationale: most MCP-server breakage in practice is import-time (missing dep, bad require path, etc.), not protocol-handshake correctness. The doctor command is a smoke test, not a conformance test. If a future Phase needs full conformance, that becomes a dedicated MCP-conformance test plan, not a doctor sub-check.

Implementation: spawn the server, listen for ANY line on stdout that parses as JSON-RPC (`{jsonrpc: "2.0", ...}` or any object the server emits during init), within a 1-second budget. On first valid JSON line → PASS with `mcp server alive in Nms`. On 1s timeout with no output → WARN `mcp server did not produce JSON-RPC output in 1s`. On spawn error / process crash → WARN with the error code. Always SIGTERM the process at the end.

The roadmap intro paragraph says "7-check" but the detailed REQUIREMENTS spec lists 8 (numbered 1-8). **This plan ships 8 checks.** Surface the prose discrepancy in SUMMARY.md for Phase 122 doc reconciliation.
</background>

    The existing hub.js file does NOT have a `fetchWithTimeout` helper, but cmdDoctor's checks 1, 2, and 8 all consume one with this exact contract: `{ ok, status, json, elapsedMs, error }`. Define it as a module-private function in `plugins/arcanon/worker/cli/hub.js`, placed near the top of the file (next to `parseArgs` at hub.js:71-92):

    ```javascript
    /**
     * Bounded fetch with timeout. Returns a normalized result object regardless
     * of success/failure so callers don't need try/catch boilerplate.
     * @param {string} url
     * @param {number} timeoutMs
     * @param {RequestInit} [opts]
     * @returns {Promise<{ok: boolean, status: number, json: any, elapsedMs: number, error: string|null}>}
     */
    async function fetchWithTimeout(url, timeoutMs, opts = {}) {
      const ctrl = new AbortController();
      const t = setTimeout(() => ctrl.abort(), timeoutMs);
      const start = Date.now();
      try {
        const res = await fetch(url, { ...opts, signal: ctrl.signal });
        const json = await res.json().catch(() => null);
        return { ok: res.ok, status: res.status, json, elapsedMs: Date.now() - start, error: null };
      } catch (e) {
        return { ok: false, status: 0, json: null, elapsedMs: Date.now() - start, error: e.message };
      } finally {
        clearTimeout(t);
      }
    }
    ```

      const serverPath = path.join(__dirname, '..', 'mcp', 'server.js');
      const t0 = Date.now();
      const result = await new Promise((resolve) => {
        const proc = spawn(process.execPath, [serverPath], {
          stdio: ['pipe', 'pipe', 'pipe'],
        });
        let buf = '';
        const timer = setTimeout(() => {
          try { proc.kill('SIGTERM'); } catch {}
          resolve({ status: 'WARN', detail: 'mcp server did not produce JSON-RPC output in 1s' });
        }, 1000);

        proc.stdout.on('data', (chunk) => {
          buf += chunk.toString();
          // Liveness probe: any valid JSON-RPC line on stdout proves the
          // server reached its message loop without crashing on import.
          // We do NOT send tools/list and do NOT enforce the handshake — that
          // is left to a future MCP conformance test. (Decision per FLAG 5
          // Option B: smoke over conformance.)
          const lines = buf.split('\n');
          for (const line of lines) {
            const trimmed = line.trim();
            if (!trimmed) continue;
            try {
              const msg = JSON.parse(trimmed);
              // Accept any object that looks like a JSON-RPC frame
              if (msg && (msg.jsonrpc === '2.0' || typeof msg.id !== 'undefined' || msg.method)) {
                clearTimeout(timer);
                try { proc.kill('SIGTERM'); } catch {}
                resolve({ status: 'PASS', detail: `mcp server alive in ${Date.now() - t0}ms` });
                return;
              }
            } catch { /* not yet a complete JSON line */ }
          }
        });

        proc.on('error', (e) => {
          clearTimeout(timer);
          resolve({ status: 'WARN', detail: `mcp spawn error: ${e.code || e.message}` });
        });

        proc.on('exit', (code, signal) => {
          // If the server exited before producing any JSON-RPC, that's a smoke fail.
          // The setTimeout handles "started but silent"; this handles "crashed on import".
          if (signal !== 'SIGTERM' && code !== 0) {
            clearTimeout(timer);
            resolve({ status: 'WARN', detail: `mcp server exited with code ${code} before producing output` });
          }
        });
      });
      return result;
    }));
