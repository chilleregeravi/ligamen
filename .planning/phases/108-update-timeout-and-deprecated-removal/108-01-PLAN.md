<objective>
Decouple `/arcanon:update --check` offline-decision from the 5-second `claude plugin marketplace update arcanon` refresh-process outcome. The mirror file on disk is the source of truth — refresh failure (timeout, transient error) is a staleness signal, not an offline signal.

**Bug today:** When `claude plugin marketplace update arcanon` takes longer than 5 seconds, `update.sh` sets `OFFLINE=true` and returns `status: "offline"` even when a fresh marketplace.json with a newer version is sitting on disk. Users get told they're offline when they actually have an upgrade waiting.

Purpose: Fix THE-1027. Trustworthy update-check is foundational for the v0.1.3 "Trust & Foundations" milestone — users need to believe the version-check status they see.

Output: Updated `scripts/update.sh` with a file-existence-based offline gate, plus three new bats tests (UPD-04/05/06) covering slow-refresh, missing-mirror, and equal-version states.
</objective>

JSON output contract for `--check` mode (must remain stable — commands/update.md parses these fields):

```typescript
type UpdateCheckResult = {
  status: "newer" | "equal" | "ahead" | "offline" | "unknown";
  installed: string;          // semver from plugin.json/package.json or "unknown"
  remote: string | null;      // semver from mirror marketplace.json or null when offline

  changelog_preview: string;  // first 2-4 bullet lines of remote CHANGELOG, or "" when not newer
};
```

Marketplace mirror file path (constructed in update.sh):
```bash
MARKETPLACE_DIR="${HOME}/.claude/plugins/marketplaces/arcanon"
MIRROR_FILE="${MARKETPLACE_DIR}/plugins/arcanon/.claude-plugin/marketplace.json"
```

Background-timer pattern in update.sh (lines 209-225) — preserve the structure, drop only the `OFFLINE=true` assignment on timeout:
```bash
{
  (claude plugin marketplace update arcanon >/dev/null 2>&1) &
  refresh_pid=$!
  elapsed=0
  while kill -0 "$refresh_pid" 2>/dev/null; do
    sleep 0.2
    elapsed=$((elapsed + 1))
    if [[ $elapsed -ge 25 ]]; then  # 25 * 0.2s = 5s
      kill -TERM "$refresh_pid" 2>/dev/null || true
      sleep 0.1
      kill -KILL "$refresh_pid" 2>/dev/null || true
      # Was: OFFLINE=true ; break  ← BUG. New: just break, no flag flip.
      break
    fi
  done
  wait "$refresh_pid" 2>/dev/null || true
} 2>/dev/null
```

Single offline gate becomes:
```bash
if [[ ! -f "$MIRROR_FILE" ]]; then
  printf '{"status":"offline","installed":"%s","remote":null,"update_available":false,"changelog_preview":""}\n' "$INSTALLED_VER"
  exit 0
fi
```

# Helper: stub the `claude` binary to sleep N seconds then exit 0
stub_slow_claude() {
  local sleep_secs="$1"
  cat &gt;"$STUB_DIR/claude" &lt;&lt;EOF
#!/bin/sh
sleep $sleep_secs
exit 0
EOF
  chmod +x "$STUB_DIR/claude"

# Helper: write the marketplace mirror file with a given version
write_mirror() {
  local version="$1"
  local mirror_dir="$TEST_HOME/.claude/plugins/marketplaces/arcanon/plugins/arcanon/.claude-plugin"

  printf '{"version":"%s","name":"arcanon"}\n' "$version" &gt; "$mirror_dir/marketplace.json"
}

# Helper: read installed version from the plugin under test
installed_version() {

}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user shell → `claude` CLI | The `claude plugin marketplace update arcanon` subprocess is invoked from a user's shell; the binary is on PATH and outside our trust boundary |
| `claude` CLI → marketplace mirror file | The CLI writes the marketplace.json; we only read it. We trust the JSON shape but validate version with `semver.valid()` before using it (existing guard at line 245 — preserved). |
| update.sh stdout → commands/update.md parser | Our JSON output is consumed by the slash-command markdown which uses `jq` to read fields. Field names and types must remain stable. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-108-01 | Tampering | marketplace.json mirror file (read by update.sh) | accept | The mirror is written by the user's own `claude plugin marketplace update`; if an attacker can write to `${HOME}/.claude/plugins/marketplaces/arcanon/...` they already have local-user code execution. The existing `semver.valid()` guard (line 245 in update.sh) prevents version-string injection into the node `-e` eval. No new tampering surface introduced by this change. |
| T-108-02 | Denial of Service | slow `claude plugin marketplace update` blocking the foreground command | mitigate | Existing 5s background timer (lines 209-225) is preserved. SIGTERM at 5s, SIGKILL at 5.1s. Refresh subprocess cannot block the user-visible `--check` for more than ~5 seconds. This change *increases* DoS resilience because we no longer fail-closed to "offline" on timeout — a slow refresh now degrades gracefully to "use the cached mirror." |
| T-108-03 | Information Disclosure | bats tests writing to `$BATS_TEST_TMPDIR` | accept | Tests use `BATS_TEST_TMPDIR` (auto-cleaned by bats), `HOME` override scoped to the test process, and a stub `claude` binary in a per-test PATH prefix. No leakage of real `$HOME/.claude/...` state into test fixtures or vice versa. |
| T-108-04 | Spoofing | stub `claude` binary in test PATH | accept | The stub is created in `$BATS_TEST_TMPDIR/stubs`, only on PATH for the duration of the test process. No production code path runs in test mode. The stub pattern matches existing bats conventions in `tests/` (e.g., `tests/worker-start.bats` uses similar PATH-override stubs for `claude`-adjacent binaries). |
</threat_model>
