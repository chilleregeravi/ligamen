<objective>
Refactor the MCP server to resolve its SQLite database per-call using pool.js, and add an optional `project` parameter to all five MCP tools so agents in any repo can query any project's graph.

Purpose: The MCP server currently opens a single DB for its lifetime using process.cwd() at startup. Any agent working in repo-B that queries the MCP server always opens repo-B's DB even when asking about repo-A — every cross-project query returns empty results or wrong data. This change makes all five tools project-aware.

Output:
- worker/db/pool.js — adds getQueryEngineByRepo(repoName); removes inline migration workaround (lines 178-202)
- worker/mcp/server.js — resolveDb(project) helper replacing module-level openDb(); optional project param on all 5 tools; db.close() calls removed
- worker/mcp/server.test.js — tests for per-call resolution and error path
</objective>

2. ADD resolveDb helper after the existing logger setup:
```javascript
function resolveDb(project) {
  if (!project) {
    const root = process.env.ALLCLEAR_PROJECT_ROOT || process.cwd();
    return getQueryEngine(root);
  }
  // Absolute path → look up by project root
  if (path.isAbsolute(project)) {
    // Security: reject path traversal
    if (project.includes('..')) return null;
    return getQueryEngine(project);
  }
  // 12-char hex hash
  if (/^[0-9a-f]{12}$/.test(project)) {
    return getQueryEngineByHash(project);
  }
  // Repo name — search all project DBs
  return getQueryEngineByRepo(project);
}
```
